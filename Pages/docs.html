The J.S.W.A.T Documentation

Introduction

Built for JS lovers, by a young, aspiring JS enthusiast, Muhammed Shaaqir Abrahams.

J.S.W.A.T is an on-going Open-Source project for the community to enjoy it's fun and easy implementation on JavaScript based websites. Built and based on the JavaScript template engine "EJS" and the scalable V8 Node.js engine.

A custom, fun, easy-to-use, free-to-use JavaScript web application starter template utilizing various technologies and libraries. J.S.W.A.T makes it easy to incorporate full stack application technologies if a simple front-end application isn't enough. Designed for any JS lover and makes it easy to implement a user-friendly, On-The-Go development environment. The template was designed with the MVC-2(Models, Views, Components & Controllers) design & development approach, making it easy to work and navigate through the project structure and implement a web application.

A complete project folder structure is set up with both "Frontend" & "Backend" directories within the main "Application" directory. Don't let the J.S.W.A.T Directory Tree scare you with a complex but efficient project structure. Loads of extra resources are included, making dependency and external library management a breeze.

NOTE: This documentation and its' guides are complete and detailed and by no means short, but definetely simple! If there is anyting not understood or should be mentioned, feel free to contact us from the Home page.

There are 2 ways to download a J.S.W.A.T Starter Template, mainly:

  1. The GitHub Template downloaded directly from the GitHub Website .
  2. The NPM Template generated directly from the NPM Tool - J.S.W.A.T-GenX .

What you should already know

This guide assumes you have the following basic background:

  • A general understanding of the Internet and the World Wide Web (WWW).
  • Good knowledge of HTML5, CSS3, JavaScript, EJS, Node.js, Express.js.
  • Good knowledge of the MVC(Model, View, Controller) development approach.
  • Good knowledge of Backend technologies, API's and RESTful services.
  • Good knowledge of version & source code control with Github/GitBash.
  • Some programming experience. If you are new to programming and web development, we have created a guided course which combines the several different web technologies & languages we use in the J.S.W.A.T Template design.

Getting Started

The J.S.W.A.T MVC-2 Directory Tree:


A detailed description of the directory tree and it's structure follows below.

The following views describe the directory tree:







Directory Tree Guide:

LEGEND:

TAG: The tag here specifies the folder or namespace in the directory tree.

DESCRIPTION: The description here specifies and provides details about the relative TAG in the directory tree.

TAG-COLOR: The color here will determine whether a TAG is:

  • Backend: YELLOW
  • Frontend: BLUE


    • BACKEND: The root/parent folder for all your backend/server-side code logic & resources.

    API: The root/parent folder for your Controllers, Renders & Routers.

  • Sub-Directory - CONTROLLERS: The folder for creating your code logic for business logic & API definitions.
  • Sub-Directory - RENDERS: The folder for creating your code logic for dynamically rendering API routes.
  • Sub-Directory - ROUTERS: The folder for creating your code logic for defining API & website routes.

    • DB: The root/parent folder for your Configs, Connection & Models.

  • Sub-Directory - CONFIGS: The folder for creating your code logic for Database configurations (NOTE: NOT FOR APP_SECRETS!).
  • Sub-Directory - CONNECTION: The folder for creating code logic for your Database connections.
  • Sub-Directory - MODELS: The folder for your creating code logic for Database schemas & models.

    • FRONTEND: The root/parent folder for all your frontend/client-side code logic & resources.

    ASSETS: The root/parent folder for your CDNX, Effects, Favicon, Images & Styles.

  • Sub-Directory - CDNX: Contains a static file with all necessary CDN libraries/dependencies for webpages.
  • Sub-Directory - Effects: Contains JS/jQuery functions/effects for webpages.
  • Sub-Directory - Favicon: Contains a default website icon(logo) for webpages.
  • Sub-Directory - Images: Contains images for use on webpages.
  • Sub-Directory - Styles: Contains CSS styles for use on webpages.

    • COMPONENTS: The root/parent folder for your Custom, J.S.W.A.T-X & Main component library.

  • Sub-Directory - Custom: Contains custom and self-created EJS components.
  • Sub-Directory - J.S.W.A.T-X: Contains J.S.W.A.T-X EJS components.
  • Sub-Directory - Main: Contains the Main EJS components.

    • SERVICES: The root/parent folder for your frontend/client-side web services & logic.

    VIEWS: The root/parent folder for your Custom & Main views.

  • Root-Directory - Main(unnamed): Contains default application views and pages.
  • Sub-Directory - Custom: Contains custom views and pages.

    Terminal/command-line navigation

    • Here are a few key commands you will be using and what they are:

    • The "npm" prefix is used to access the npm package manager instance.

    • The "-i" prefix is used to install a package/dependency.

    • The "-s" prefix is used to save a package/dependency when installing one.

    • The complete command for installing a package/dependency is "npm -i -s [packageName]".

    • The "start" prefix is used to start a project through node.js and npm on your local server.

    • The complete command for starting your project on your local server is "npm start".

    • There are more commands and prefixes to use on the npm website, for now, that's all we need.

    Build & Complie

    • Here we will use NPM to inject our dependencies into our project.

    • It is not required that you install any of the dependencies in your "package.json" file, Node.js & NPM will scan this automatically when we inject the dependencies.

    • Make sure you are in the current directory as followed in the download section:
      "C:\Users\kryptiq-labs.dev\Desktop\J.S.W.A.T\Application" .

    • Where "kryptiq-labs.dev" will be your own PC_Name.

    • Where "Desktop" will be your desired save location.

    • Where "Application" will be your default/custom app name.

    • Where "C:\Users\kryptiq-labs.dev\Desktop\J.S.W.A.T\Application" will be your root project directory.

    Let's start the dependency-build:

    1. In your terminal, as shown below, enter the command "npm install"

      2. C:\Users\kryptiq-labs.dev\Desktop\J.S.W.A.T\Application>npm install

    2. The build should take a few minutes, let's create our application-secrets file so we can deploy our project to the local server:
      • In your root directory, that is "C:\Users\kryptiq-labs.dev\Desktop\J.S.W.A.T\Application", close all folders so the explorer is cleared up, then create a file called "config.env". This file will hold your application secrets and securely load them without leaving them vulnerable.
      • This "config.env" file, if you are not already aware, will not be pushed to your Git repository because it has been mentioned in your ".gitignore" file. For that reason Git will not push your secrets and they will stay secure, in your own hands!
      • Now navigate to the file called "APP-SECRETS.txt" and cut & paste it into your "config.env" file.
      • J.S.W.A.T has already resolved the secrets file, if you require it in other files, you can import it.
      • By now the dependency injection process has completed and we should now have two new items, a "node_modules" folder and a "package-lock.json". We should now be ready for deployment!

    Deploy & Open

    It's time to run this project!

    By now, you should have followed the guide successfully and your project explorer should look similar to the below image.

    If you are getting any errors, go back to the top and start again, CAREFULLY!

    Our last two commands we will be using are "npm start" and a *Wildcard ("Ctrl" + "C").
    These are to start and kill our server whenever it is required. Do take note that a monitoring tool has already been installed as a dependency, called "nodemon". This restarts your server whenerv any changes take place in your IDE/code.

    Start your local server with the command: "npm start".
    This will deploy the application to your local server on default PORT that is defined by J.S.W.A.T e.g. PORT:3000
